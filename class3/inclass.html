In-class Exercise: JavaScript

Part 1: Change some settings on a Leaflet map

  1. Remix this Glitch site in your account.
  2. The map is currently showing the whole world at once but the data is mostly on the eastern US coast. Let's recenter the map.
  3. In the script.js file on line 4 there should be a line like this one: 
    var map = L.map('map').setView([34.03, -82.20], 5);
    On this line:
    1. 34.03 is the center latitude,
    2. -82.20 is the center longitude, and
    3. 5 is the zoom level.
  4. Try editing the zoom and see what changes on the map.
  5. Try editing the latitude and longitude. If you want to be more precise, try using bboxfinder to find a helpful center. (Keep an eye on the order of the latitude and longitude though!)

Part 2: Change the basemap

  1. Continuing with your code from the previous part, let's change the basemap. On line 7 there is a url like this: 
    https://stamen-tiles.a.ssl.fastly.net/toner/{z}/{x}/{y}.png
  2. Check out the OpenStreetMap wiki page on tiles and pick one to replace the above URL with. For example, maybe I want a hillshade basemap like this: 
    http://c.tiles.wmflabs.org/hillshading/${z}/${x}/${y}.png
    I'll delete the old URL in my code and paste the new URL in. Make sure you remove the dollar signs ($) though! 
    http://c.tiles.wmflabs.org/hillshading/{z}/{x}/{y}.png

Part 3: Load your own data

  1. Remix this Glitch project into your Glitch account.
  2. Go to gist.github.com and find the map data you created last week in class. If you can't find the data or didn't create any, make some simple data now with geojson.io.
  3. Download the data from Gist by clicking the Raw button and saving the resulting page.
  4. Upload the data to your Glitch assets folder (as you would with an image).
  5. Copy the URL for the uploaded data and paste it in line 12 of the JavaScript, replacing the URL that is currently there.
  6. View the resulting page and confirm that your data is loaded on the map.
