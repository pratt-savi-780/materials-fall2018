In-class Exercise: Data, More JavaScript

Part 1: Filtering data in the NYC Open Data API

  1. Remix this Glitch site in your account or continue working from the site from the previous example.
  2. Open the JavaScript file.
  3. Change the borough filter to QUEENS and confirm that the map data is updated.
  4. Keeping that borough filter, add a filter for complaint_type.
  5. Remove the borough and complaint_type filters from your URL (you can comment the code out if you want to hold on to it).
  6. Refer to the API documentation for the dataset and for queries more generally.
  7. Use in in a $where filter to find complains in two or three boroughs at once.
  8. Use another function to do more filtering.

Part 2: Filtering data on an event

  1. Remix this Glitch project into your Glitch account.
  2. Confirm that the Brooklyn button works.
  3. Implement the Queens button—make it so that when the Queens button is clicked, data is loaded for Queens.

Part 3: Dropdown events

  1. Remix this Glitch project into your Glitch account.
  2. Confirm that the dropdown works for Brooklyn.
  3. Change the HTML for the dropdown (<select>) to add the other boroughs.

Part 4: Text input events

  1. Remix this Glitch project into your Glitch account.
  2. Confirm that the text input works.
  3. The text input currently searches within complaint_type. Change loadData to search on some other field and/or some other dataset.

Part 5: forEach

  1. Create a new Glitch project that is a webpage.
  2. In the JavaScript file, create an array of numbers. For example: 
    var allTheNumbers = [1, 2, 3, 4.1, 10000000];
  3. Use forEach to log out each value in the array. For example: 
    allTheNumbers.forEach(function (number) {
  // Do something with number here
});

Part 6: Checkbox events

  1. Remix this Glitch project into your Glitch account.
  2. Add checkboxes for the other boroughs. You should only have to change the HTML to make this work.
  3. If you have time, change the HTML and loadData to filter by some of the values in complaint_type.
