In-class Exercise: Data, More JavaScript

Part 1: Adding your own data to Mapbox

  1. Continue with your style from Parts 1 and 2 from Tuesday.
  2. Select the water layer and click the Select data tab.
  3. At what zoom level do rivers become visible? If it helps, find the Mississippi or another prominent river and zoom out until it disappears.
  4. Let's say we are mapping rivers want them to be visible when zoomed out more. Download Natural Earth's rivers data and go to Tilesets in your Mapbox account and upload the zip file you downloaded. This will take a minute or two to upload and process.
  5. Go back to editing your style.
  6. Click + Layer, then click the button next to Source. Find and click on the rivers under Unused sources. The name should start with ne_10m....
  7. Set the Zoom max so that this layer does not overlap with the other rivers layer.
  8. Add a filter to only select features with scalerank less than 5—we only want to show the more prominent rivers.
  9. Click Style and style the layer.

Part 2: Put a site on GitHub Pages

In this exercise, we'll put a Glitch site online with GitHub Pages. This process is long but we outline the steps here and you can refer to tonight's slides for images showing the process.

  1. Open GitHub.
  2. Click on the + at the top right of the page, then New Repository.
  3. Give the repository a name and check Initialize this repository with a README.
  4. Under the repository's Settings, scroll down to GitHub Pages select master under Source.
  5. Open any Glitch site in your account.
  6. Click on the project name at the top left of the page.
  7. Click Advanced Options.
  8. Click Export to GitHub. You may have to give Glitch access first.
  9. In the dialog that opens, type your GitHub username, a / and the GitHub repository name from earlier. For example, my GitHub username is ebrelsford. If my repository is leaflet-map-class10, I would type ebrelsford/leaflet-map-class10.
  10. Go back to GitHub and open the repository that you created earlier. Click Branch, then glitch.
  11. Click New Pull Request.
  12. Click Create pull request.
  13. On the next page, click Merge pull request.

Part 3: Making web pages on your own computer

Get some practice making web pages on your computer, outside of Glitch.

  1. Open the same Glitch site in your account from the previous exercise.
  2. Click on the project name at the top left of the page.
  3. Click Advanced Options.
  4. Click Download Project. The project will be compressed in a file like a zip file—open it.
  5. Install a code editor if you don't already have one. Try Atom, Brackets, or Sublime Text.
  6. Open the downloaded Glitch project in the code editor and make some changes to it.
  7. In GitHub, open the repository you created in the previous exercise.
  8. Drag and drop all of the files from your project onto the repository page and commit the changes.

Part 4: Responsive CSS

Try using media queries to make styles that are dependent on the width of the user's screen.

  1. Create a new Glitch site or work from an existing one.
  2. Using the code in this site's CSS, use media queries to change the appearance of your site on smaller screens.
