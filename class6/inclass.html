In-class Exercise: More JavaScript

Part 1: Functions

  1. Start a blank Glitch site in your account.
  2. In script.js, write a function that takes a number as the parameter and returns the number two times the parameter. For example, if your function is called with 4 the function should return 8.
  3. Confirm that your function works by viewing the resulting site (Show Live) and calling the function through the console in developer tools.

Part 2: Styling features

This is a continuation of Part 3 from last class.

  1. Remix this Glitch site in your account.
  2. Open the Leaflet documentation. For this exercise you'll be most interested in GeoJSON and Path options
  3. Style the polygons (council districts) by adding an options parameter to L.geoJson and creating a style function. Use this Glitch site for assistance as needed.
  4. Style the points (311 complaints) by adding an options parameter to L.geoJson and creating pointToLayer and style functions as is done in this Glitch site.

Part 3: Conditionally styling features

  1. Remix this Glitch site in your account.
  2. The points are currently styled differently where status is Closed. Instead style the points where complaint_type is HEATING.
  3. Try styling another value of complaint_type differently.

Part 4: A zoom out button

  1. Remix this Glitch site in your account.
  2. The zoom in button should work on the resulting page.
  3. Implement JavaScript for the zoom out button—it should be very similar to the code for the zoom in button.

Part 5: Add data with a button

  1. Remix this Glitch project into your Glitch account.
  2. Use Show Live and open developer tools on the page.
  3. Call loadData to load the data on the map.
  4. Back in the JavaScript, add an event listener that calls loadData when the load data button is clicked.

Part 6: Listening for Leaflet events

  1. Remix this Glitch project into your Glitch account.
  2. In the JavaScript, listen for more map events and log out the name of the event. For example, copy 
    map.on('moveend', function () {
  console.log('moveend');
});
    paste those lines (so the moveend listener still exists) and change moveend in the new listener, like this: 
    map.on('click', function () {
  console.log('click');
});
    Don't forget to change what is logged out, too, so you can see which event happened.
  3. Do the above process until you have five different map events. Pay special attention to:
    • click
    • dblclick
    • mousedown
    • mouseup
    • move
    How do the above events differ from each other?
